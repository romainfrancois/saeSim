Start simulation

Usage

sim(x, ...)


"sim"(x, ..., R = 1, parallel = FALSE, path = NULL)

Arguments

x
a sim_setup or data.frame
...
simulation components; If parallel = TRUE arguments passed to mclapply, see details.
R
number of repetitions.
parallel
if FALSE lapply is used; if TRUE a 'plug-and-play' version of mclapply is used; see details.
path
optional path in which the simulation results can be saved.

Start simulation

Start simulation

Value

The return value is always a list. The elements are the resulting data.frames of each simulation run.

Description

This function can be applied to a sim_setup. It will start the simulation. Use the printing method as long as you are testing the scenario.

Details

The backend for parallel computation is very experimental. You can find the code and documentation on www.github.com/wahani/parallelTools. In Windows parallelization is build around clusterApply. If you are not using Windows the function mclapply is used. See the parameters of mclapply how to control the parallelization. Parallizing trivial tasks in Windows will result in wasted time. Also working with large data.frames will be inefficient in Windows. Use the path to store the simulation results to a directory. This may be a good idea for long running simulations and for those using large data.frames. Use sim_read_data, to read load them again.

Examples

setup <- sim_base_lm()
resultList <- sim(setup, R = 1)

# Will return a data frame
dat <- sim(sim_base() %>% sim_gen_x() %>% sim_gen_e())
